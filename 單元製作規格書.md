# 學習聯絡簿 — 單元製作規格書 (v1)

給 Claude Code 用。目標：照這份規格，把「國一上」剩下的練習單元一個一個做出來，全部符合同一套接口，能被主儀表板（`index.html`）自動掛載、記錄進度與錯題。

**範本檔案**：`signed-muldiv.html`（正負數乘除）。所有新單元都以它為基礎，**只換出題邏輯與解說**，其餘（CSS、版面、鍵盤、hook）保持不變。

---

## 0. 一句話原則

簡單、夠用、有深度。介面跟範本一致；出題正確可驗算；解說逐步、講重點；不做多餘功能。全繁體中文，適合國中生。

---

## 1. 硬性接口契約（每個單元都必須做到）

1. **單一 HTML 檔**，無外部相依（除了範本已引用的 Google Fonts）。可單獨用瀏覽器開，也可被主程式用 `<iframe>` 嵌入。
2. **讀網址參數 `?level`**：值為 `basic|adv|mix`，載入時套用對應難度並反白對應難度鍵。範本結尾已有這段，原封保留。
3. **每次作答檢查時，用 postMessage 回報父視窗**（單獨開啟時不受影響）：
   ```js
   if (window.parent !== window) {
     window.parent.postMessage({type:'unit-progress', unitDone:1, unitCorrect: correct?1:0}, '*');
     if (!correct) {
       window.parent.postMessage({type:'unit-mistake', q: qText(), ans: String(正解), given: 使用者答案字串}, '*');
     }
   }
   ```
   - `qText()`：回傳「這一題的純文字題目」字串（給錯題本顯示用），例：`"(−6) × (+4) ="`。
   - `ans`：正解的字串。
   - `given`：使用者輸入的字串（負號用 `−`）。
4. **難度三級**：`基礎(basic) / 進階(adv) / 混合(mix)`，用範本上方的 `.lv` 鍵切換。
5. **檔名**必須等於 `units.json` 裡該單元的 `file` 欄位（見第 6 節對照表）。
6. **反向「遊戲指令」介面（供外層 RPG 對題目下技能）**：單元除了往外回報，還要能**接收父視窗指令**。單獨用瀏覽器開時（沒有父視窗送指令）一切照舊。標準介面如下：
   - **載入完成時**，回報自己支援哪些技能，讓遊戲決定要不要顯示技能鈕：
     ```js
     if (window.parent !== window)
       window.parent.postMessage({type:'unit-ready', supports:['hint']}, '*'); // 選擇題單元再加 'fifty'
     ```
   - **監聽父視窗指令**：
     ```js
     window.addEventListener('message', e => {
       const d = e.data; if (!d || d.type !== 'unit-command') return;
       if (d.cmd === 'hint')  showHint();      // 顯示提示：給關鍵概念/公式/第一步，不直接給答案
       if (d.cmd === 'fifty') eliminateOne();  // 選擇題：反灰並停用一個「非正解」選項；數字題：給弱提示
     });
     ```
   - `showHint()`：把該題已有的 `buildTip()`（口訣/關鍵概念）或 `buildSteps()` 的第一步，顯示在題目下方（塞進動態建立的 `#game-hint`），一次一題；換下一題時清掉。**不得洩漏答案**。
   - `eliminateOne()`：**選擇題單元**才實作——隨機把一個「非正解」選項變灰、`disabled`（正解永遠不被消掉）；**數字輸入單元**可先只呼叫 `showHint()`（弱提示），並在 `supports` 只放 `['hint']`。
   - 實作方式：用**獨立的第二段 `<script>`（IIFE）** 掛在檔尾，只讀不改原本邏輯（可安全批次套用到所有單元）；換題清提示的做法是包一層原本的 `next()`／`nextProblem()`。
7. **自我描述 meta（讓 `units.json` 能自動生成，不用手改）**：每個單元 HTML 的 `<head>` 內要放一段**純資料**的 JSON（單獨開啟不受影響、不會執行）：
   ```html
   <script type="application/json" id="unit-meta">
   {
     "id": "linear-eq",
     "title": "一元一次方程式",
     "subject": "數學",
     "grade": "國一",
     "term": "上",
     "chapter": 8,
     "tag": "方程式",
     "ready": true
   }
   </script>
   ```
   - `id`：唯一代碼（＝**檔名去掉 `.html`** 最保險）。
   - `grade`：`國一 / 國二 / 國三`；`term`：`上 / 下`（兩者合起來＝舊的 `"國一上"`）。
   - `subject`：`數學 / 生物 / 國文 / 英文 / 歷史 / 地理 / 公民`（要跟 `units.json` 的 `subjects` 對得上）。
   - `chapter`：**這科這學期的第幾章**（數字，決定預設排序）。
   - `tag`：小分類標籤；`ready`：做好了沒（`true/false`）。
   - **`units.json` 由這些 meta 自動產生**（掃所有單元 HTML → `tools/gen_units.py`），不要再手改 `units.json`。改單元屬性就改它自己 HTML 裡的這段 meta。

> 第 1～5 點是「能不能接進系統」的關鍵；第 6 點是「能不能被 RPG 當技能操作」的關鍵；第 7 點是「能不能自動生成 `units.json`」的關鍵，皆不可更動格式。

## 2. 範本結構：哪些保留、哪些替換

打開 `signed-muldiv.html`：

**原封保留（不要改）**
- `<style>` 整段（版面、鍵盤、`.eq/.ans/.explain/.steps/.tip` 樣式、正負號顏色 `.pos`藍/`.neg`橘）。
- `<body>` 的外殼：header、`.levels` 難度鍵、`.card`、`.pad` 數字鍵盤、`.go` 按鈕。
- JS 中的這些「固定件」：鍵盤事件、`.go` 事件、`.lv` 切換、實體鍵盤、`?level` 解析、`next()`、以及第 1 節的 postMessage hook。

**替換成該單元的邏輯**
- `makeProblem(level)` 與底下的 `gen*()`：出題。
- `eqHTML()` / `render()`：把題目畫到 `#eq`。
- `check()` 裡的「正解比對」與「解說產生」。
- `buildSteps()` / `buildTip()`：逐步解說與提示。
- `qText()`：純文字題目。
- header 的 `<h1>` 改成單元名稱。

**可調整**
- header 副標、答案位數上限（`entry.replace('-','').length < N`，依答案大小設 N）。

## 3. 輸入模式（依單元選用）

大多數單元用**模式 A**。只有少數需要 B/C。

- **模式 A — 整數鍵盤（範本現況）**：數字 0–9、`±`、`⌫`。答案是單一整數（可負）。**優先用這個**：能設計成單一整數答案的題型，就設計成單一整數。
- **模式 B — 比大小三鍵**：把數字鍵盤換成三顆大按鈕 `<`、`=`、`>`。作答值為所選符號字串。用於「兩數比大小」。
- **模式 C — 分數輸入**：一列 `±` 鍵 + 「分子」欄 + 分母欄（兩個 number 輸入）。判定時，把使用者答案與正解**都化成最簡分數**再比較（`sign, num, den`；整數視為 `den=1`）。只有 `分數四則` 用。

> 若某題型自然會產生分數答案（如除不盡），優先改題型讓答案為整數；真的必要（分數單元）才用模式 C。

## 4. 通用出題規則

- **除法必整除**、**方程式必為整數解**：用「先定答案、再反推題目」的方式產生，避免出現除不盡或非整數解。
- **控制數字大小**：答案位數不超過鍵盤可輸入上限（建議 ≤ 6 位）；乘積、次方要限制範圍。
- 負底數次方：偶次冪為正、奇次冪為負，要算對。
- 正負號在題目與解說中用顏色標示（`.pos` 藍色代表正、`.neg` 橘色代表負），沿用範本的 `termHTML`/`fmt` 寫法。
- 每種難度至少 2~3 種題型變化，避免重複感。

## 5. 測試要求（每個單元都要做）

做完一個單元，**先把出題函式抽出來用 Node 跑 ≥ 10000 題自動驗算**，確認正解計算與（除法/方程式）約束全部成立，再收工。範例骨架：

```js
let bad=0;
for(let i=0;i<10000;i++){
  const p = genSomething();
  const expect = 用獨立方式重算(p);   // 用與出題不同的方式再算一次
  if(expect !== p.answer) bad++;
  // 若有整除/整數解約束，一併檢查
}
console.log(bad===0 ? "OK" : "BAD="+bad);
```

## 7. 各單元規格

以下每個單元，`基礎/進階/混合` 對應 `basic/adv/mix`。

### num-line — 數與數線（含絕對值）
- **重點**：絕對值、正負數比大小、數線上兩點距離。
- **基礎**：求絕對值 `|a|`，a 為 ±整數(1..20)。→ 模式 A，整數答案。
- **進階**：兩數比大小，顯示 `a ▢ b`，作答填 `<`/`=`/`>`。→ **模式 B**。
- **混合**：絕對值 + 比大小 + 兩點距離 `|a−b|`（答案整數，模式 A；比大小題用模式 B）。
- **解說**：絕對值＝與原點的距離，永遠 ≥ 0；比大小＝數線上愈右邊愈大；距離＝`|a−b|`。
- **qText 例**：`"|−7| ="`、`"−3 ▢ +5"`。
- 註：此單元同時有 A/B 兩種輸入，依題型動態切換鍵盤區。

### sci-notation — 指數記法與科學記號
- **重點**：次方運算、科學記號 ↔ 一般記法。
- **基礎**：次方求值 `aⁿ`（含 `(−a)ⁿ`）；限制大小（如 2^≤10、3^≤6、底數≤9、n≤4）。→ 模式 A。
- **進階**：科學記號→一般數（`a×10ⁿ`，a 為一位小數 1.0~9.9，n 1..6，答案為整數）；或 一般數→求指數 n（答案 n）。→ 模式 A（答案永遠是單一整數）。
- **混合**：以上混合。
- **解說**：`aⁿ`＝a 連乘 n 次；科學記號係數 1≤|係數|<10，×10 的次方＝小數點移動位數。
- 註：科學記號一律讓答案是「一般數」或「指數」等單一整數，不用結構輸入。

### prime-factor — 質因數分解
- **重點**：質數、質因數分解、因數個數。（全用整數答案，避免集合輸入。）
- **基礎**：給合數 N(≤50)，求「最小質因數」。
- **進階**：給 N(≤100)，求「因數個數」＝各質因數(指數+1)的乘積。
- **混合**：加「最大質因數」。（皆整數，模式 A。）
- **解說**：先顯示 N 的質因數分解 `2^a×3^b…`，再據此說明答案。
- **規則**：N 必為合數；分解與因數個數用程式算對。

### gcd-lcm — 最大公因數與最小公倍數
- **重點**：HCF、LCM。
- **基礎**：兩數求最大公因數（兩數 4..48）。
- **進階**：兩數求最小公倍數（兩數 2..24，LCM 控制在數百內）。
- **混合**：HCF/LCM 混合，可含三數。
- **解說**：列兩數質因數分解；HCF＝取公同質因數的最小指數之積、LCM＝取所有質因數的最大指數之積。
- **規則**：gcd 用輾轉相除；lcm = a·b/gcd。模式 A。

### fraction-ops — 分數的四則運算
- **重點**：分數（含正負）加減乘除、最簡分數。
- **基礎**：同分母或簡單異分母加減；或分數乘法。
- **進階**：異分母加減（通分）、分數除法（顛倒相乘）、含負分數。
- **混合**：四則混合（一次一個運算，不做長串）。
- **輸入**：**模式 C**（正負 + 分子 + 分母）。判定時兩邊都約分成最簡再比。
- **解說**：通分→分子運算→約分；乘法分子×分子、分母×分母；除法乘倒數。
- **規則**：先算出正解並約分；分母不得為 0。

### exponent-law — 指數律
- **重點**：`aᵐ·aⁿ=aᵐ⁺ⁿ`、`aᵐ÷aⁿ=aᵐ⁻ⁿ`、`(aᵐ)ⁿ=aᵐⁿ`。
- **基礎**：同底相乘/相除，求「結果的指數」（底數固定顯示，答案是指數，整數，可負）。
- **進階**：次方的次方 `(aᵐ)ⁿ`；或連續運算。
- **混合**：以上混合；小數字時偶爾求「值」。
- **輸入**：模式 A，答案為指數（除法可能出現負指數，允許 ±）。
- **解說**：相乘→指數相加、相除→相減、次方→相乘。

### algebra-expr — 代數式與化簡
- **重點**：合併同類項、代入求值。
- **基礎**：代入求值，單一未知數（給 x＝?，求 `ax+b`）。→ 整數答案。
- **進階**：合併同類項係數（如 `3x+2x−x`＝`▢x`，只填係數 4）；或代入含負數/兩項。
- **混合**：代入求值（含乘除）＋係數合併。
- **輸入**：模式 A。合併同類項時題目顯示「▢x」，學生只填係數。
- **解說**：同類項係數相加；代入＝把數字代進去算。
- **規則**：答案保持整數。

### linear-eq — 一元一次方程式
- **重點**：解一元一次方程式。
- **基礎**：`ax+b=c`，整數解（先定 x 為 ±整數，反推 b、c）。求 x。
- **進階**：`ax+b=cx+d`（兩邊有 x）或 `a(x+b)=c`（含括號），整數解。
- **混合**：以上混合。
- **輸入**：模式 A，x 可負（±）。
- **解說**：移項→合併→兩邊除以 x 的係數。
- **規則**：由整數 x 反推係數，確保整數解。

### linear-eq-app — 一元一次方程式的應用
- **重點**：用一元一次方程式解文字題。
- **題型模板**（隨機挑）：某數問題、連續整數、年齡、購買總價、周長。
  - **基礎**：單步（某數的 a 倍加 b 等於 c，求某數）。
  - **進階**：兩步、連續整數之和、年齡差問題。
  - **混合**：多模板隨機。
- **輸入**：模式 A，整數答案。
- **解說**：設未知數→列出方程式→解。
- **規則**：先定整數答案，反推題目數字，保證乾淨；文字生活化、繁中、適合國中生。

## 8. 完成後如何上線

1. 把做好的 `xxx.html` 放進 GitHub repo 根目錄（跟 `index.html` 同層）。
2. 在 `units.json` 把該單元的 `"ready": false` 改成 `"ready": true`。
3. Commit。主程式會自動在「單元」清單開放它，可勾選、可設每日題數與難度、答錯會進錯題本。

> 不需要動 Supabase。內容全在 repo，帳號的進度與錯題不受影響。
