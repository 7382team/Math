# 國中勇者 · 遊戲設計文件（GDD v0.1）

> 活文件。把「學習聯絡簿」的題庫，包成一個打怪升級的 RPG。邊做邊改。
> 搭配原型：`game.html`（打第一隻怪，已可玩）。

---

## 0. 一句話定位

**把每天的練習，變成一場一層一層攻略的紙上冒險。** 答題＝戰鬥，學會＝變強，錯題＝要收服的心魔。

## 1. 核心原則（設計時的定海神針）

1. **同一顆引擎，兩張皮。** 現有的練習單元（`signed-muldiv.html` 等）是「內容引擎」；`index.html`（聯絡簿）是**紀律外殼**，`game.html`（RPG）是**動機外殼**。兩個外殼吃同一批單元，單元一行都不用改。
2. **不亂答題。** 優先用「數字輸入」（天生防猜），答錯有真代價，重考用**同型新題**而非換選項位置。
3. **複習＝報仇＋收集，不是重做作業。** 錯題變心魔、會回來、收服給最好的獎勵。
4. **有得有失，但失有上限。** 撤退不沒收素材；最慘只是白跑一趟。定位大眾 → 易學難精，預設寬容、困難當選配。
5. **先好玩再美術。** 用佔位圖把核心迴圈做爽，再砸美術。方向早定、產出後補。

## 2. 架構：一個內容引擎 + 兩個外殼

```
15+ 個練習單元 (HTML)                聯絡簿外殼  index.html（現有）
  每個都會發 postMessage  ──►  作答事件匯流排  ──►
  ?level=basic|adv|mix              RPG 外殼    game.html（新）
```

**單元 → 外殼的既有契約（不要改）：**
- 讀網址參數 `?level`。
- 每次作答：`postMessage({type:'unit-progress', unitDone, unitCorrect})`。
- 答錯：`postMessage({type:'unit-mistake', q, ans, given, steps, tip})`。

對 RPG 而言：`unit-progress{correct:1}` ＝ **對魔物造成傷害**；`unit-mistake` ＝ **你挨打＋生成心魔（進錯題本）**；單元裡的 `streak` ＝ **連段**。

> 原型 `game.html` 為了做出整合式的戰鬥畫面（怪＋題＋鍵盤同框），**內嵌了一份正負數乘除的出題邏輯**；正式版改為用 iframe 載入單元、靠 postMessage 驅動，即可支援全部單元。

## 3. 雙軸進度（化解「現實進度 vs 遊戲成長」的矛盾）

把進度**拆成兩條各自獨立的軸**，就不糾結了：

| 軸 | 是什麼 | 由誰決定 | 對到現有 |
|---|---|---|---|
| **A 內容範圍** | 哪些副本/樓層「開著」 | 老師/家長（現實進度） | `selected_units`、`course_by_subject` |
| **B 角色成長** | 等級、裝備、村莊、外觀、素材 | 遊戲內努力賺 | 新增（與課程無關，永遠能漲） |

孩子隨時感受到 B 在漲（RPG 爽感），即使 A 被外部鎖住。**學校還沒教新章時**，B 軸讓他有事做：心魔副本複習、衝精熟度、日常探索刷素材、攻城事件、排行榜。

## 4. 城鎮塔（＝A 軸的控制台，而且九成已存在）

- **塔的每一層 ＝ 一個學期**；老師教到哪、塔就開到哪一層。點亮一層 → 該學期科目區域開放。
- **家長設外框、孩子在框內自由選**（對上「想刷什麼都隨機、都在範圍內」）。
- **省事關鍵**：家長設定**繼續留在聯絡簿**（已做好），`game.html` 只負責**讀 `settings` → 畫出世界**。塔在遊戲裡主要是「看進度＋選今天打哪關」，家長那間房上個 PIN 即可。

## 5. 世界結構

- **村莊／根據地** ＝ 當前學期。周邊傳送門 ＝ 已解鎖的科目。
- **副本** ＝ 科目（例：正負數）。
- **樓層** ＝ 章節，順序 ＝ `course.sequence`（例：康軒數學 11 個單元 ＝ 11 層）。一個月推一層。
- **BOSS 房** ＝ 每個副本結尾的精熟關。

## 6. 任務類型（核心固定，換皮保鮮）

同一套答題戰鬥，換不同包裝，不做新機制：

1. **標準討伐**：清 N 隻小怪 → BOSS。（基本款）
2. **菁英狩獵**：一隻血厚菁英（題多），稀有掉落。
3. **限時連段**：時間內連對衝分（＝ `streak`）。
4. **攻城／防守（隨機事件）**：BOSS 來襲，一波波題目擊退。**題目從「錯題本＋舊章節」抽 → 同時是間隔複習**，也給「今天有事件、想上線」的理由。
5. **日常探索**：不設固定目標，怪物/素材從**目前解鎖範圍內隨機**抽（mix 跨單元），純放鬆刷。

## 7. 戰鬥系統（核心迴圈）

**魔物 ＝ 一組題目，血量 ＝ 題數**（小怪 3，BOSS 6）。單題迴圈：

- **答對（第一次）** → 魔物 −1 血、連段 +1、跳傷害數字、命中震動、機率掉素材。
- **答錯** → 你被考倒挨打（−1 血）、連段歸零、魔物**當場講解**；接著**立刻用講解重考一題同型的（防禦）**：
  - 防禦答對 → **擋下攻擊**（不再挨打），但**魔物不掉血、照樣帶著這題型逃跑**（＝之後還要再答一次 → 加深記憶）。
  - 防禦答錯 → 再挨一下。
- **所有題答完，魔物還有血** → 小怪**帶著你答錯的題型逃跑**；點「追擊」→ 只考那題型（**重新生成同型新題**）直到 0 血。**BOSS 不逃跑，改成把你震開**（本次失敗、回一點血、你的錯題變成 BOSS 招式）。
- **魔物 0 血** → 擊倒，掉素材（**一次全對＝稀有 ✦星塵**）。**你 0 血** → 撤退（**素材保留**、該樓層重打）。

**防作弊 & 對齊學習：**
- 優先數字輸入；選擇題只給概念題，且重考**重生成**不換位置。
- 掉落綁「第一次答對／解決逃跑怪」，不綁「作答次數」。
- BOSS ＝ mix 綜合，靠猜一定撐不到最後 → 天然的「學會才過關」。

## 8. 錯題 ＝ 心魔（間隔重複）

- 答錯的題型 → 一隻**心魔**在世界遊蕩，照排程回來：**隔 1 → 3 → 7 天**再現。
- 收服（第一次答對）→ 最稀有的獎勵／變寵物。
- **心魔副本**：全由你的錯題組成，通關給頂級寶（＝把最好的獎勵綁在最該做的複習上）。
- **精熟門檻**：錯題累積太多，下一層的門被「封住」，逼你先清錯題。
- 建議升級：訂正要「**不同天答對 2 次**」才畢業，比一次訂正更牢固。

## 9. 經濟與建設

- 素材分兩類：**強化素材**（角色基礎值）與**建築/外觀素材**（村莊隨學期升級變華麗）。
- 得失表（大眾寬容版）：

| 情況 | 得 | 失 |
|---|---|---|
| 第一次答對 | 滿額傷害·掉寶率高·連段+1 | — |
| 防禦重考答對 | 擋下攻擊·小素材 | 魔物仍帶血逃跑（要再答） |
| 答錯 | — | 扣血·連段歸零·生心魔 |
| HP 歸零→撤退 | **素材全保留** | 該樓層重打 |

## 10. 美術方向

- **2D 紙上冒險 / 立體書（papercraft）**，延伸 `index.html` 的印章紙感色票。怪物＝紙片小獸，村莊＝聯絡簿長出來的世界。
- 可愛靠**一致方向 + 手感 juice**（命中震動、粒子、Q 彈縮放、傷害數字），不靠 3D。
- 一個人做：itch.io / Kenney 素材包或 AI 生立繪，先佔位、後替換。

## 11. 技術

- **單檔 HTML/CSS/JS，無 build**（沿用現有風格，`python -m http.server` 或直接開檔）。
- 跨平台走 2D（Canvas/DOM），手機優先；戰鬥時 UI 切滿版卡片模式。
- **Phase 1 不需要遊戲引擎**；戰鬥＝單元 iframe 外包一層 HUD。長出重動作層再考慮 Phaser。
- 連線分階段見路線圖；WebRTC 只在 Phase 3。

## 12. 資料模型（沿用現有 + 少量新增）

**沿用**：`settings`（selected_units / course_by_subject / daily_goals / rest_days）、`activity`（做題數/月曆）、`mistakes`（錯題本＝心魔來源）。

**新增（Phase 1 可先放 localStorage，之後上 Supabase）：**
- `inventory`：素材與道具數量。
- `buildings`：村莊建設/外觀狀態。
- `character`：等級、HP、裝備。
- `monster_review`：每個錯題型的下次出現時間（間隔重複排程）。

## 13. 路線圖

- **Phase 1 — 單機 RPG 外殼**：村莊→副本→樓層→答題戰鬥→掉素材→蓋東西。複用全部單元。**（原型已起步：`game.html`）**
- **Phase 2 — 非同步社交**：好友進度、排行榜、下戰帖。用現有 Supabase，零即時連線。
- **Phase 3 — 即時雙人＋語音**：WebRTC。答題是回合制，不需毫秒同步（共享血量、輪流出手）。語音可先用 Discord 代替。

## 14. 原型現況（game.html）

**已做**：整合式戰鬥畫面（紙感＋加減小獸）、血量＝題數、答對砍刀/連段/傷害數字/命中震動、答錯→講解→防禦重考、帶血逃跑→追擊、BOSS（綜合連續運算）、撤退保留素材、一次全對給星塵、素材計數、鍵盤＋實體鍵盤輸入。

**還沒做**：村莊/副本/樓層地圖、以 iframe+postMessage 接真正的單元（目前內嵌正負數乘除）、心魔間隔重複排程、建設/外觀、存檔（Supabase/localStorage）、社交與連線。
